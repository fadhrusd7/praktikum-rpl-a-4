# Backlog


## Must-have

- **Laporan isu lingkungan:** User dapat membuat laporan dengan mengisi informasi seperti judul, deskripsi, lokasi, dan foto (opsional). Admin dapat memverifikasi laporan.
- **Melihat laporan di peta:** User dapat melihat laporan yang telah dibuat dalam bentuk peta sehingga lokasi permasalahan dapat diketahui dengan jelas. Admin dapat melihat laporan yang masuk.
- **Registrasi dan login user:** User dan admin.

## Should-have

Fitur ini penting namun bisa dikembangkan setelah MVP selesai.

- **Mengecek status laporan:** User bisa lihat perkembangan laporan, seperti status pending, diproses, atau verifikasi.
- **Dashboard laporan:** Admin punya dashboard untuk lihat semua laporan yang masuk sehingga dapat melakukan monitoring secara keseluruhan.
- **Update status laporan:** Admin dapat memperbarui status laporan agar user mengetahui perkembangan tindak lanjut dari laporannya.
- **Statistik ringkas di dashboard admin:** jumlah laporan per kategori, per status, dan tren mingguan.

## Could-have

Fitur opsional:

- Sistem memberikan notifikasi kepada user saat status laporan berubah.
- Laporan dapat di filter berdasarkan kategori (misal sampah, jalan rusak, dll).
- User dapat melihat riwayat laporan yang pernah dibuat.
- Mode laporan anonim.

## Won't-have

Fitur tidak menjadi prioritas dalam pengembangan tahap awal.

- Sistem otomatis berbasis AI untuk validasi laporan.
- Fitur komentar antar pengguna.
- Fitur AI yang akan memblur wajah orang yang terdapat pada foto untuk privasi.
